# CuentaClara - Project Vision

## Problem

When friends or coworkers organize shared events, one person usually pays first and then has to collect money from everyone.

This creates several problems:

* people forget to pay
* screenshots are sent through WhatsApp and get lost
* the organizer has to manually check Yape, Plin or bank transfers
* it is hard to know how much has been collected
* it is hard to know who still owes money
* the organizer carries most of the burden

## Product Idea

CuentaClara helps organizers manage shared payments.

The organizer creates an event and shares a link.

Participants open the link, identify themselves, upload payment evidence, and the system validates the receipt using AI.

## Target Users

Primary user:

* event organizer

Secondary user:

* participant

## Use Cases

* football matches
* BBQs
* birthday parties
* shared gifts
* trips
* beach house rentals
* sport field reservations
* coworker events

## MVP Goal

Build the smallest useful version where:

* organizer creates an event
* app generates a shareable link
* participant uploads payment evidence
* AI validates payment evidence
* organizer sees collected and pending amounts

## Success Criteria

The organizer should be able to answer:

* How much was paid?
* How much is missing?
* Who uploaded evidence?
* Which payments were valid?
* Which payments need review?
* Who should receive a reminder?
